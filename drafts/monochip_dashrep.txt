heading-begin
The Revolution that will End Computer Central Processing Units
heading-end

paragraph-begin
Smartphones, and computers, waste huge amounts of time and electricity.
sentence-next
Why?
sentence-next
All the calculations are done in central processing units.
sentence-next
As the word italics-begin central italics-end suggests,
all the numbers being processed have to wait their turn
getting to and from the central processing unit.
sentence-next
Yes there are lots of ways to speed up that transfer of data.
sentence-next
Yet that bottleneck is still there.
paragraph-end

paragraph-begin
The secret to making mobile devices that run much, much faster,
with less waste of battery power,
is to distribute simple processing units throughout
each and every computer chip.
sentence-next
This means each small group of memory locations will have their own
simple processing unit.
paragraph-end

paragraph-begin
It's like ...

paragraph-end

subheading-begin
Serial processing units, not central processing units
subheading-end



subheading-begin
Each Memory Word Gets Its Own Serial Processing Unit
subheading-end




subheading-begin
Chip defect tolerance
subheading-end

paragraph-begin
The chip's huge number of simple processing units allows lots of them to have chip defects.
sentence-next
The runtime compiler can avoid using the units that have a defect.
sentence-next
This tolerance will dramaticlly reduce the manufacturing cost.
paragraph-end


subheading-begin
Programming a serial processing unit
subheading-end

paragraph-begin
Instead of instructions, a string of bits is sent to the input side of the serial processing unit, and those bits select which inputs are supplied to that serial processing unit.
paragraph-end

paragraph-begin
Each serial processing unit is a one-bit full adder. sentence-next It has an A and B input and a carry input. sentence-next It has a main one-bit output and a carry output. sentence-next 

paragraph-begin
Based on what is supplied to the A, B, and carry inputs, the main one-bit output of the full adder can calculate these one-bit results:
paragraph-end

list-begin
ADD
list-next
OR
list-next
XOR
list-next
NOT
list-end

paragraph-begin
Using the above calculations can yield the following numbers:
paragraph-end


paragraph-begin
As explained above, a string of bits select which one-bit data inputs are supplied to that serial processing unit. sentence-next Here are all the possible inputs. sentence-next Only some of them are available to only some serial processing unites (each of which is a single-bit full adder).
paragraph-end

list-begin
from_adder_above
list-next
from_adder_left
list-next
from_adder_right
list-next
from_adder_left_pairwise_N sentence-next where N can be 0 through 31
list-next
from_zero
list-next
from_one
list-next
from_one_initial sentence-next which increments the first bit within a number, and afterward is zero
list-next
from_size_bits_8
list-next
from_size_bits_16
list-next
from_size_bits_32
list-next
from_size_bits_64
list-end

paragraph-begin
The "from_size_bits" inputs allow numeric "words" to be of different sizes. sentence-next Specifically 8 bits, 16 bits, 32 bits, and the default size of 64 bits. sentence-next Basically this input specifies a path that skips over unused bits in that 64-bit "word" storage area. sentence-next The option from_size_bits_64 is only needed if the default word length is 128 bits.
paragraph-end

paragraph-begin
The 32 pairwise inputs are used for multiplication using the method described below. sentence-next This input option is available in only some columns (if data flow is top to bottom) or only some rows (if data flow is left to right).
paragraph-end

paragraph-begin
With this simple data processing it is possible to do integer addition, integer subtraction, integer multiplication, and integer division.
paragraph-end

paragraph-begin
When decimal numbers are needed, either in the compiler or at runtime, the decimal numbers can be scaled and offset to fit between zero and the largest integer, then after calculating using those integer numbers, the integers can be converted back into decimal numbers.
paragraph-end

paragraph-begin
When floating-point numbers are used, either the compiler or the runtime code can manage the sign (plus or minus) and the exponential portion. sentence-next Addition, subtraction, multiplication, and division are done using just the integer portion. sentence-next Then the three parts (sign, mantissa, and exponent) are combined into the standard single floating-point representation.
paragraph-end

subheading-begin
Increment
subheading-end

paragraph-begin
Incrementing of a number A is done by running its bits through one processing cell (full adder) where it is added to number B which is just a single one bit as the first bit, and all other bits of B are zero. sentence-next The output carry bit is supplied to the input carry bit for each next bit in number A.
paragraph-end


subheading-begin
Addition
subheading-end


subheading-begin
Subtraction
subheading-end

paragraph-begin
Subtraction has three steps. sentence-next It is done in three processing cells that are linked in sequence. sentence-next Assume the calculation is A minus B. sentence-next The three steps are:
paragraph-end

list-begin
In step one processing cell: sentence-next Convert B into "NOT B", which is the "ones complement." sentence-next This means each 0 becomes 1, and each 1 becomes 0. sentence-next This can be done by selecting carry to be zero and selecting the other full-adder input to be one.
list-next
In step two processing cell: sentence-next Increment (add one to) NOT B as described in the "increment" section. sentence-next This is done by selecting the second full-adder input to be one during the least significant bit, and then zero for all following bits.
list-next
In step three processing cell: sentence-next Add the incremented NOT B to operand A. sentence-next Ignore the carry from the most significant bit. sentence-next The result is the difference.
list-end


subheading-begin
Decrement
subheading-end

paragraph-begin
Decrementing of a number A is done by running its bits through one processing cell (full adder) where it is added to number B which is all ones for all bits. sentence-next The output carry bit is supplied to the input carry bit for each next bit in number A.
paragraph-end


subheading-begin
Multiplication
subheading-end

paragraph-begin
Multiplication is done using the from_adder_left_pairwise_N lines as inputs to a column of serial processing units. sentence-next The calculated bits are sent to adjacent serial processing units that add these calculated bits. sentence-next Between each bit of the first (A) operand, a shift downward in operand A aligns the data for the next addition.
paragraph-end


subheading-begin
Division
subheading-end

paragraph-begin
Division starts with calculating the complement of the second operand, which is B. sentence-next Then the complemented operand B is added to the "remainder" being calculated, and also added to a possibly temporary "remainder". sentence-next If the result of the possibly temporary remainder yields a carry bit that indicates overflow, the non-modified remainder is used for the next subtraction. sentence-next If instead there is no carry, the possibly temporary remainder is used as the new remainder. sentence-next The bits of the division result is simply the sequence of bits indicating whether the addition caused a carry or did not cause a carry.
paragraph-end


subheading-begin
Data transfers
subheading-end


paragraph-begin
All data transfers copy the data without erasing the content of the source location, and without saving the previous content of the destination location.
paragraph-end

paragraph-begin
A data transfer specifies the source cluster and the source cluster offset, or the target cluster and the target cluster offset.
paragraph-end

paragraph-end
If power consumption should be minimized, the chip type is capacitor-based "latches" similar to dynamic memory chips.
sentence-next
This dramatically reduces transistor count because a latch requires many transistors.
sentence-next
In contrast, there are just two transistors between each capacitor.
sentence-next
Those transistors "amplify" the capacitor state at the input and charge or discharge the capacitor at the output.
sentence-next
(This might slightly complicate the compiler because it becomes responsible for transferring a capacitor's state before it loses its charge, but that's easy to handle because not all the groups of 16 registers are always being used for storage and processing.)
paragraph-begin

paragraph-begin
Reminder: For CMOS, favor two 2-input AND gates going into OR gate
paragraph-end
